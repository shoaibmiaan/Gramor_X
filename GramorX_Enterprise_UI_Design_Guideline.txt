# 1. Design Philosophy

GramorX UI must feel:

‚úÖ Educational but premium
‚úÖ Calm and distraction-free
‚úÖ Accessible by default
‚úÖ Consistent across all pages
‚úÖ Enterprise SaaS quality (Not startup chaos)

### Core Principles

1. **Consistency over creativity**
2. **Reuse before redesign**
3. **Clarity before decoration**
4. **Accessibility is mandatory**
5. **System > Individual Page**

# 2. Design System Architecture (Project Truth)

| Components      | `design-system/`      | Core UI primitives      |
| Layout Shells   | `layouts/`            | Page structure          |
| Storybook       | `stories/`            | Component reference     |

üö® Never invent styles outside this system.

# 6. Color Usage (Token Driven)

Designers must use semantic meaning.

| Meaning    | Usage        |
| ---------- | ------------ |
| Primary    | Main actions |
| Success    | Completion   |
| Warning    | Attention    |

‚ùå Never pick colors manually from picker.

# 7. Component Usage Rules

Use components from:

```

```

Includes:

* Checkbox
* Radio
* Select
* Toggle

### Golden Rule

> If component exists ‚Üí reuse it
> If slightly different ‚Üí extend it
> Never redesign from scratch.

# 8. Card Design Standard

All cards must include:

‚úÖ Padding: 16‚Äì24px
‚úÖ Radius from token
‚úÖ Soft shadow only
‚úÖ Clear hierarchy

Structure:

```

```

---

# 9. Buttons

### Types Allowed

| Type      | Usage            |
| --------- | ---------------- |
| Primary   | One per section  |
| Secondary | Supporting       |
| Ghost     | Low emphasis     |
| Danger    | Destructive only |

Rules:

* Never more than **1 primary button per area**
* Minimum height: 40px

---

# 10. Forms & Inputs

Use consistent flow:

```
Label
Input
Helper text
Error message
```

### Validation

* Error appears instantly
* Red + message
* Never color-only feedback

‚úÖ Color contrast AA minimum
‚úÖ Keyboard navigation works
‚úÖ Focus visible always

# 13. Premium UI Rules

Premium visuals live in:

```
styles/premium.css
premium-ui/
```

Premium = refinement, NOT redesign.

Allowed:

* subtle gradients
* elevation
* accent highlights

Not allowed:

* new layout structures.

---

# 14. Navigation Consistency

Defined in:

```
config/navigation.ts
```

Designers must NOT:

* change nav order randomly
* rename items visually without product approval

# 18. Do & Don't

## ‚úÖ DO

* Extend patterns
* Keep calm UI
* Reduce visual noise
* Align spacing precisely

## ‚ùå DON'T

* Invent new button styles
* Mix spacing scales
* Add random shadows
* Change colors manually

---

# 19. Enterprise Quality Standard

A page is considered **production-ready** only if:

* Looks identical in spacing rhythm to other pages
* No unique styling logic
* Components reusable elsewhere
* Accessible without mouse
* Feels part of one product

---

# 20. Designer Workflow

1. Check existing component
2. Check Storybook
3. Design using tokens
4. Validate accessibility
5. Submit with states
6. Developer maps to existing component
